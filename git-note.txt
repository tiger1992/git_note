
git对待数据的方式：快照流。建立分支时候的命名以实现特定功能特征来命名。

1、在工作目录中修改文件

2、暂存文件，将文件的快照放入暂存区域【git add】

3、提交更新，找到暂存区域的文件，将快照永久性存储到 Git 仓库目录【commit -m "提交备忘录"】

4、使用命令git add filename，注意，可反复多次使用，添加多个文件；只有在暂暂缓区的内容commit才会被提交到仓库。
   git add 命令使用文件或目录的路径作为参数；如果参数是目录的路径，该命令将递归地跟踪该目录下的所有文件

5、使用命令git commit -m "提交备忘录"

6、每次准备提交前，先用 git status 查看是否都已暂存起来了，然后再运行提交命令 git commit -m " "

7、git diff 本身只显示尚未暂存的改动

8、退出按q。
